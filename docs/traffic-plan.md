# Organic Traffic Plan

This document is for internal planning only. It is not a public website route.

## Principles

- Help first, link second.
- Do not buy fake traffic, fake reviews, fake backlinks, or fake testimonials.
- Do not use link farms, comment spam, automated outreach, or misleading claims.
- Do not post direct links where a community discourages self-promotion.
- Prefer profile-based discovery when direct linking would be unwelcome.

## Sharing Without Direct Links

Answer the question fully in the platform where it was asked. Mention the method, formula, or checklist directly in the answer. If a link is appropriate under the community rules, add it only when it clearly helps. If direct links are discouraged, let the profile link do the discovery work.

## Profile Link Strategy

Use one clean profile link to `https://www.freelanceworktools.com`. Keep bios simple: free calculators and practical guides for freelance rates, billable hours, project pricing, invoices, and remote work costs. Avoid exaggerated claims.

## Quora Strategy

Look for questions about freelance rates, hourly pricing, fixed project pricing, billable hours, invoices, and remote work expenses. Write useful answers with a practical framework. Avoid pasting the same answer repeatedly. Use the profile link rather than forcing a link into every answer.

## Dev.to Strategy

Publish educational posts for developers and technical freelancers, such as calculating billable capacity, pricing fixed-scope projects, or comparing salary and contract rates. Include formulas and examples. Link only when context allows and the post stands alone without the link.

## Reddit No-Link Discussion Strategy

Participate in discussions without dropping links. Share formulas, examples, and decision frameworks directly in the comment. If someone asks for the tool, say it is linked in the profile only when that fits the community rules. Do not evade moderation.

## LinkedIn Strategy

Post short practical lessons: one pricing mistake, one calculation example, one billable-hours insight, or one invoice term explanation. Avoid hard selling. Use comments to answer questions with more detail.

## Medium Strategy

Republish or adapt deeper guides as standalone educational articles. Use canonical links only if appropriate. Keep the article useful even when readers never click through.

## Weekly Publishing Schedule

- Monday: publish one practical pricing or capacity insight on LinkedIn.
- Tuesday: answer two relevant Quora questions.
- Wednesday: draft or refresh one guide based on search intent.
- Thursday: participate in one community discussion without direct linking.
- Friday: publish one deeper Dev.to or Medium article when there is a complete topic.
- Weekend: review Google Search Console queries and update the roadmap.

## Avoid Spam Rules

- Never post the same comment across multiple threads.
- Never pretend to be an unrelated user recommending the site.
- Never create fake reviews or testimonials.
- Never buy traffic, backlinks, or social engagement.
- Never use doorway pages or thin country pages.
- Never imply the calculators provide legal, tax, or financial advice.
